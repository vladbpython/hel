## [1.1.0]

### Added

- **Module `hel::helper::batch`** a set of four helpers for channel drainage
  batches, symmetrical for sync and async, through closures (without traits).
  Recipient type is displayed at the call peer, covers all three channels
  (`shard_key`, `round_robin`, `spsc`):
  -`drain_batch` โ element by element processing, `handler: FnMut(T, &mut O)`;
  -`drain_batch_sink` - gives ownership of the entire batch to the sink (via
    `std::mem::take`) for calculations on an array at once (sum/min/max/SIMD);
  -`drain_batch_async` โ async mirror of `drain_batch`;
  -`drain_batch_async_sink` โ async mirror of `drain_batch_sink`; battery
    ownership is transferred back and forth via future (for sending a batch via
    network/in the database with one `await`).
  All helpers respect the drainโcheck invariant: the buffer is processed before the flag is checked disconnect, since the last batch and closing the channel come in one
  calling `recv_batch`. The buffer is reused between iterations (emptied along the length maintaining capacity)


### Fixed

- **`Sender::send_batch_async`: added fast-path via `push_batch`.**
  Previously, the async version of batch sending ignored `push_batch` and sent the elements
  one at a time via `send_async().await` in a loop (with preliminary
  `buf.reverse()`). For SPSC this meant that batch submission worked.
  **slower**element by element. Now, as in synchronous `send_batch_impl`,
  `push_batch` is tried first (one tail publication and one
  `notify_receivers` for the entire batch), and only when the channel is full element by element fallback is performed, preserving the FIFO.

-**`SeqInner` has become a single cross-architectural implementation (x86 + ARM).**
  Previously, there were two separate platform implementations of the internal
  channel state; separate `Inner` for x86 has been removed and its behavior
  merged into `SeqInner`, which now works on both architectures.
  This eliminates code duplication and a single point for platform-specific
  constants (spin/yield), simplifying support.

-**Benchmarks have been converted to methodology with delivery verification and clean
  measurement.** Data pools (`Trade`, Copy, `&'static str` fields) are allocated
  once per group outside the dimension and are reused via `Arc`. Everyone
  run checks the checksum (`ฮฃ t`) via `assert_eq!` loss or
  Duplicate messages drop the benchmark, rather than distort the number. Async froze
  built on the starting `AsyncBarrier`: runtime is created once, timer covers only transfer, without creating runtime and spawning tasks.
  The oversubscription group is presented separately as incomparable with the main ones.

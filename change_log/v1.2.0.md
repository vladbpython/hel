## [1.2.0]

### Added

- **`ShardGroup`** — third keyed routing mode for `mpmc`: many keys → few shards
  by an explicit user defined map, unlike random hash distribution in `shard_key`. 
  Group is chosen by you, not by hash.
  - `from_groups(&[&[&str]])` — group i → shard i, shard count from `groups.len()`;
  - `from_map(HashMap<String, usize>, num_groups)` — ready map + group count,
    indices normalized via `% n`;
  - `shard_group(ShardGroupCase)` — single constructor dispatching to the above.
  - `handle(key) -> Option<SymbolHandle>` — resolve key → shard index ONCE at
    subscription; hot path sends by handle (~0.6 ns, no hash, no lookup);
  - `shard_handle() -> ShardHandle` — standalone resolver sharing the route via
    `Arc` (cheap clone), hand to producers without the full `ShardGroup`.
  - Single send by handle: `try_send` / `send` / `send_timeout` / `send_async`;
  - Batch by handle (one group → one shard): `try_send_batch` / `send_batch` /
    `send_batch_async`, FIFO preserved under back pressure;
  Receiver is unchanged.

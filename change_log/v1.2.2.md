## [1.2.2]

### Modified

- **`ShardGroup`** — batch send methods now auto route a mixed batch across
  shards by `key_fn`, instead of sending a pre grouped batch by a single handle.
  - `try_send_batch`, `send_batch`, `send_batch_timeout`, `send_batch_async` now
    take `(buf, key_fn)`: a batch of DIFFERENT keys is split into per shard
    groups via the explicit route map, then each group is sent to its shard.
  - Unregistered keys are returned as unused in `buf` (never misrouted); on a
    shard error `buf` also holds all unsent elements (per key FIFO preserved).
  - Signature aligned with `shard_key` batch methods
    (`key_fn: impl for<'k> Fn(&'k T) -> &'k str`), so element type now carries
    the key (e.g. `(String, u64)`).

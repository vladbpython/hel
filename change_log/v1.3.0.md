## [1.3.0]

### Added
 
- **Elastic consumer pool (`pool` feature).** A new opt in module that runs an
  adaptive set of consumers over any MPMC channel. Fully compatible with all
  three channel constructors — `round_robin`, `shard_key`, and `shard_group`
  by consuming their `into_receivers()` output. Enable with
  `features = ["pool"]`; the bare channel remains dependency free by default.
  - `sync_pool(...)` and `async_pool(async_runtime, ...)` — construct a pool over a set
    of receivers with a handler.
  - Runtime: the async pool spawns and sleeps through the user supplied
    `AsyncRuntime` adapter (implement it for tokio, async-std, etc.). No async
    runtime is pulled in by the crate itself.
  - **Handlers.** `PerItem` (process one element at a time) and `Batch` (process
    a whole slice at once), each in sync and async form. Custom handlers can
    implement `SyncHandler` / `AsyncHandler` directly (e.g. a database sink that
    collects rejected rows).
  - **Elastic scaling.** The pool grows and shrinks its active consumer count
    based on channel fill: fast jump up under load, gradual step down when idle.
    Configurable via `Config` (`min`/`max` consumers, scale thresholds,
    `batch_size`, sample interval).
  - **Three shutdown modes.**
    - `drop(tx)` + `wait_stopping()` — auto drain: processes everything already
      sent, then stops (zero loss).
    - `get_signal_stop().stop()` + `wait_stopping()` — signal cancellation:
      finishes the in flight batch, skips the remaining queue.
    - `stop_and_wait()` — cancel and join in a single call.
  - Order preservation per key with `shard_key` / `shard_group` (a symbol always
    lands on the same shard and is consumed in FIFO order).
- **`queued()`** on channel inners — returns the approximate number of buffered
  elements (`tail - head`, saturating).

### Fixed
- **Inline slot storage.** Channel slot arrays are now stored inline
  (`[Slot<T>; CAP]`) instead of behind a `Box`. This removes one level of
  indirection on the hot path (`Arc -> slots` instead of `Arc -> Box -> slots`),
  improving MPMC throughput by roughly a third on the benchmarked workload.
- **Heap placement constructors.** Channel inners are now constructed directly
  in their `Arc` allocation via `Arc::new_uninit` + placement, so large inline
  slot arrays are never materialized on the stack. This keeps the throughput
  win of inline storage while avoiding stackoverflow risk for large `CAP`.
  Verified under Miri (initialization correctness, no leaks, no stack overflow).

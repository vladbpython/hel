## [1.3.2]

### Fixed

- **Your payload type no longer has to implement `Debug`.** The send error types
  that carry the item back (`ShardTrySendError`, `ShardSendError`,
  `ShardKeyTrySendError`, `ShardKeySendError`, `ShardAsyncSendError`,
  `ShardKeyAsyncSendError`) derived `Debug`, and that quietly required `Debug` on
  the item as well. So sending a type that could not be printed meant you could
  not print the error, call `unwrap`, or use it with `?` in most setups. The
  errors now print the shard, the key, and what went wrong, without touching the
  item. The item itself is handed back to you inside the error, which is where
  you wanted it anyway. Nothing changes for types that already implement `Debug
